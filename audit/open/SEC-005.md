# Security Finding: SEC-005

## Unsafe Type Casting in DI Container

**Severity:** HIGH  
**CVSS Score:** 7.5  
**CWE:** CWE-763 (Release of Invalid Pointer or Reference)  
**Status:** 🔴 **OPEN**

---

## Description

The DI container uses unsafe type casting with `voidptr` and `unsafe` blocks, bypassing V's type safety.

### Vulnerable Code

```v
// src/services/registry.v

pub fn (sr ServiceRegistry) get_config() ?&ConfigService {
    instance := sr.container.resolve('config') or {
        return error('ConfigService not registered')
    }
    unsafe {
        return &ConfigService(instance)  // TYPE SAFETY BYPASSED
    }
}

pub fn (sr ServiceRegistry) get_logger() ?&LoggerService {
    instance := sr.container.resolve('logger') or {
        return error('LoggerService not registered')
    }
    unsafe {
        return &LoggerService(instance)  // TYPE SAFETY BYPASSED
    }
}
```

### DI Container Storage

```v
// src/di.v

pub struct ServiceDescriptor {
pub mut:
    name          string
    lifecycle     ServiceLifecycle
    instance      voidptr  // Raw pointer - no type information
    factory       fn () voidptr
    initialized   bool
}
```

---

## Impact

### Type Safety
- **High:** No compile-time type checking
- Runtime crashes if wrong type is resolved
- No IDE autocomplete or refactoring support

### Memory Safety
- **Medium:** Potential for memory corruption
- Dangling pointers if service disposed
- No ownership tracking

### Maintainability
- **High:** Difficult to refactor safely
- Type changes require manual audit
- Tests may not catch type mismatches

---

## Attack Scenarios

### Scenario 1: Type Confusion
```
1. Developer registers ServiceA as 'my_service'
2. Later code expects ServiceB and calls get_serviceB()
3. Unsafe cast succeeds but returns wrong type
4. Method calls access wrong memory offsets
5. Crash or undefined behavior
```

### Scenario 2: Use After Free
```
1. Service resolved and stored
2. Container disposed (frees service)
3. Stored reference still used
4. Access to freed memory
5. Crash or security vulnerability
```

## Remediation

### Option 1: Use V Generics (Recommended)

```v
// Generic DI container
pub struct Container {
pub mut:
    descriptors     map[string]ServiceDescriptor
}

pub struct ServiceDescriptor[T] {
pub mut:
    instance      T
    factory       fn () T
    lifecycle     ServiceLifecycle
}

// Type-safe registration
pub fn (mut c Container) register_singleton[T](name string, instance T) {
    c.descriptors[name] = ServiceDescriptor[T]{
        instance: instance
        lifecycle: .singleton
    }
}

// Type-safe resolution
pub fn (mut c Container) resolve[T](name string) !T {
    descriptor := c.descriptors[name] or {
        return error('Service not found')
    }
    
    // Type check at runtime
    if descriptor is ServiceDescriptor[T] {
        return descriptor.instance
    }
    
    return error('Type mismatch for service')
}
```

### Option 2: Add Runtime Type Checking

```v
pub struct ServiceDescriptor {
pub mut:
    name          string
    type_name     string  // Store type information
    instance      voidptr
    lifecycle     ServiceLifecycle
}

pub fn (mut c Container) resolve[T](name string, expected_type string) !T {
    descriptor := c.descriptors[name] or {
        return error('Service not found')
    }
    
    // Runtime type check
    if descriptor.type_name != expected_type {
        return error('Type mismatch: expected ${expected_type}, got ${descriptor.type_name}')
    }
    
    unsafe {
        return T(descriptor.instance)
    }
}

// Usage with type name
logger := container.resolve[LoggerService]('logger', 'LoggerService') or {
    panic(it.message)
}
```

### Option 3: Use Interface Types

```v
pub interface IService {
    name() string
    init() bool
    dispose()
}

pub struct ServiceDescriptor {
pub mut:
    instance &IService  // Interface pointer instead of voidptr
    lifecycle ServiceLifecycle
}

// Type-safe through interface
pub fn (mut c Container) resolve(name string) !&IService {
    descriptor := c.descriptors[name] or {
        return error('Service not found')
    }
    return descriptor.instance
}

// Cast to concrete type with type assertion
logger := container.resolve('logger') or { return }
if logger is &LoggerService {
    logger.info('message')
}
```

## Testing

### Unit Tests for Type Safety

```v
fn test_di_type_safety() {
    mut container := new_container()
    
    // Register service
    mut logger := new_logger_service()
    container.register_singleton('logger', &logger)
    
    // Resolve with correct type
    resolved := container.resolve('logger') or {
        assert false
        return
    }
    
    // Verify type
    assert resolved is &LoggerService
    
    // Wrong type should fail
    wrong := container.resolve_as('logger', 'ConfigService')
    assert wrong is error
}
```
