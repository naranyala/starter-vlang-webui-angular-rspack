# Architecture Finding: ARC-004

## Missing Circuit Breaker Pattern

**Severity:** MEDIUM  
**Impact:** Cascading failures, no graceful degradation  
**Status:** 🔴 **OPEN**

---

## Description

No failure isolation for external dependencies. Failed services don't fail fast.

### Current Pattern

```v
w.bind('getSystemInfo', fn (e &ui.Event) string {
    info := system.get_system_info()  // Can hang indefinitely
    return json.encode(info)
})
```

---

## Impact

- No failure isolation
- Slow dependencies block all requests
- No graceful degradation
- No automatic recovery

---

## Remediation

Implement circuit breaker:

```v
pub struct CircuitBreaker {
pub mut:
    failure_threshold int
    success_threshold int
    timeout_ms int
    state CircuitState
    failures int
    last_failure_time u64
}

pub enum CircuitState {
    closed      // Normal operation
    open        // Failing fast
    half_open   // Testing recovery
}

pub fn (mut cb CircuitBreaker) execute[T](fn () T) !T {
    if cb.state == .open {
        if time.now().unix() - cb.last_failure_time < cb.timeout_ms {
            return error('Circuit is open')
        }
        cb.state = .half_open
    }

    result := fn() or {
        cb.failures++
        if cb.failures >= cb.failure_threshold {
            cb.state = .open
        }
        return error
    }

    cb.successes++
    if cb.successes >= cb.success_threshold {
        cb.state = .closed
        cb.failures = 0
    }

    return result
}

// Usage
w.bind('getSystemInfo', fn (e &ui.Event) string {
    result := circuit_breaker.execute(fn () string {
        info := system.get_system_info()
        return json.encode(info)
    })
    
    return result or {
        return '{"error":"Service temporarily unavailable"}'
    }
})
```
